HOW TO USE:

Step 1: Launch GUI
   python gui_traffic_simulation.py

Step 2: Set Duration
   Enter number of minutes (1-60) in "Durasi" spinner

Step 3: Start Simulation
   Click "Mulai Simulasi" (green button)

Step 4: Watch Violations
   - Violations appear in table below
   - Dashboard updates in real-time
   - Pelanggaran count increases
   - Denda (IDR) accumulates

Step 5: View Details
   - Click "Lihat" button for any violation
   - Dialog shows:
     * Plat Nomor (License Plate)
     * Wilayah (Region/Province)
     * Tipe Kendaraan (Vehicle Type)
     * Nama (Owner Name)
     * NIK (16-digit ID)
     * Tempat Tinggal (Residence)
     * Status STNK (Registration Status)
     * Status SIM (Driving License Status)
     * Kecepatan (Speed Detected)
     * Denda (Fine in IDR)

Step 6: Stop Simulation
   Click "Hentikan Simulasi" (red button)

Step 7: Clear Data
   Click "Hapus Data" to reset for new run

OWNER INFORMATION DISPLAY:

Each violation shows owner details:

Name (Nama): 
- Full Indonesian name
- Example: "Budi Wijaya", "Siti Nurhaliza", "Ahmad Rahman"

ID (NIK):
- 16-digit Nomor Induk Kependudukan
- Example: "3275891501823719"
- Format: 16 digits as Indonesian ID

Residence (Tempat Tinggal):
- Province/region derived from plate code
- Example: "Bandung (Jawa Barat)", "Jakarta (DKI)"

Registration Status (Status STNK):
- Active: Vehicle registration valid
- Non-Active: Vehicle registration expired (red color)

Driving License Status (Status SIM):
- Active: Driving license valid
- Expired: Driving license expired (red color)

RUPIAH CURRENCY IMPLEMENTATION:

Exchange Rate: 1 USD = 15,500 IDR

Fine Levels (Indonesian Traffic Fines):
- 76-90 km/h: $100 = Rp 1,550,000
- 91-110 km/h: $200 = Rp 3,100,000
- 111-130 km/h: $500 = Rp 7,750,000
- 131+ km/h: $1,000 = Rp 15,500,000

Currency Display:
- Format: "Rp X,XXX,XXX" (Rupiah with thousand separators)
- Location: Dashboard, violation table, detail dialogs
- Both USD and IDR shown in detail view

Dashboard Shows:
- "Total Denda (IDR): Rp X,XXX,XXX"
- Updated in real-time as violations occur
- Display in dark red for visibility

TECHNICAL ARCHITECTURE:

GUI Framework: PyQt5 (Qt5 for Python)

Main Classes:
1. SignalEmitter(QObject)
   - Emits signals from simulation threads
   - Enables thread-safe GUI updates

2. SimulationWorker(QThread)
   - Runs simulation in background thread
   - Prevents UI blocking
   - Emits real-time updates

3. ViolationDetailDialog(QDialog)
   - Shows detailed violation information
   - Professional dialog layout
   - All owner and registration details

4. TrafficSimulationGUI(QMainWindow)
   - Main application window
   - Left panel: Dashboard and controls
   - Right panel: Violations table

Threading Model:
- Main Thread: GUI and user interaction
- Worker Thread: Simulation execution
- Signal/Slot: Thread-safe communication

Real-time Updates:
- Dashboard updates every 2 seconds
- Violations appear immediately when detected
- Table auto-refreshes
- Statistics accumulate
